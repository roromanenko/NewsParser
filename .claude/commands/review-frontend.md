Use the **react-ui-reviewer** agent (file:
`.claude/agents/react-ui-reviewer.md`) to review the following scope:
$ARGUMENTS

If no scope was provided, ask me which folder or files to review before
starting. Suggest sensible defaults based on `reviews/PROJECT_MAP.md`, for
example:
- `UI/src/features/` — feature modules (articles, auth, sources, users)
- `UI/src/features/<feature>/hooks/` — React Query hooks
- `UI/src/components/` — shared and UI components
- `UI/src/store/` — Zustand stores
- `UI/src/router/` — route definitions and guards (`ProtectedRoute`,
  `AdminRoute`)
- `UI/src/lib/` — Axios instance, utilities
- `UI/src/context/` and `UI/src/hooks/` — cross-feature hooks/contexts

Requirements:
1. Read `CLAUDE.md`, `UI/CLAUDE.md`, and `docs/reviews/PROJECT_MAP.md` first.
2. Limit the review to `.ts`, `.tsx`, `.css` files and any module-level
   config (`vite.config.ts`, `tsconfig*.json`, `eslint.config.*`) inside
   the given scope.
3. **Do not review files under `UI/src/api/generated/`** — they are
   auto-generated by `npm run generate-api` and must be regenerated, not
   edited. Only flag if they have been hand-edited or if surrounding
   integration code misuses them.
4. Append findings to `reviews/frontend-findings.md` in the format defined
   in the agent's instructions. Do not overwrite existing findings.
5. When done, print a one-paragraph summary: total findings by severity,
   and the top 3 issues I should look at first.
6. Do not modify any source code.
